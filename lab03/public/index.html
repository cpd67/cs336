Lab 3

Answers for the lab 3 exercises:
  • 3.1

    • The purpose of Node.js is to provide a Javascript runtime for your application. The purpose of Express is to provide a flexible web framework for your application. Node.js provides the runtime, while Express provides a web framework (module) that you load into your application. Their purposes are similar in that they both provide more funtionality to your web application.

    • The purpose of the packages.json is to provide information about your application. This includes the name, version, scripts, and any dependencies that your application relies on (such as Express) and what version those dependencies are.

  • 3.2

    • They are static in the sense that they do not change. The server doesn't need to do any work in order to create them. It simply sends the file over and it is display on the page.

    • When you first connect to the port number that the server is listening on, then it will serve up the "Hello World!" message. However, if you access a different file (index.html, for example), then the server will send over the file so that it can be displayed onto the webpage. It will no longer server the "Hello World" message at that point because it is serving the file that you want to access.
